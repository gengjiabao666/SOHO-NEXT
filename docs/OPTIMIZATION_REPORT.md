# Echotik Collector 优化报告

## 执行时间
2026-03-06

## 问题诊断

### 主要问题
根据日志分析（`logs/echotik_2026-03-06.log`），发现以下问题：

1. **侧边栏导航点击失败**
   - 症状：登录成功后，点击「选品」和「小店」一级菜单失败
   - 日志：`点击失败: 侧边栏一级: 选品`
   - 影响：导致所有下载任务失败

2. **可能原因分析**
   - 页面加载时间不足（原等待10秒，Echotik首页需约30秒完全加载）
   - 弹窗处理不完善（可能有残留遮挡）
   - 选择器策略单一（只尝试了6种选择器）
   - 缺少元素可见性和可点击性检查

## 优化方案

### 1. 增强侧边栏导航逻辑 (`browser/downloader.py`)

#### 新增功能
- **导航前截图**：记录导航前页面状态
- **侧边栏加载检测**：等待侧边栏容器元素出现
- **增加等待时间**：从800ms增加到1200ms和2000ms
- **增强版点击函数**：`_click_by_text_enhanced()`

#### 新增 `_click_by_text_enhanced()` 函数特性
- 支持20+种选择器策略
- 精确匹配 + 包含匹配
- 侧边栏特定选择器（`[class*='sidebar']`, `nav`, `aside`等）
- 元素可见性检查
- 自动滚动到元素
- 详细的调试日志
- 失败时分析页面文本内容

### 2. 优化登录流程 (`browser/session.py`)

#### 页面加载等待
- 从10秒增加到15秒
- 更符合Echotik首页实际加载时间

#### 弹窗处理增强
- 增加2秒等待让弹窗有时间出现
- 新增更多弹窗选择器（9种）
- 添加可见性检查
- 增加调试日志
- 失败时不中断流程

### 3. 日志系统优化 (`utils/logger.py`)

#### 新增功能
- 支持 DEBUG 级别日志
- DEBUG 日志只在需要时输出
- 保持终端输出简洁

### 4. 新增调试工具

#### `debug_page.py` - 页面内容调试工具
功能：
- 自动登录
- 完整页面截图
- 打印所有可见文本（前100行）
- 查找侧边栏相关文本
- 检查侧边栏元素

使用方法：
```bash
python debug_page.py
```

#### `test_navigation.py` - 导航功能测试工具
功能：
- 测试多种选择器策略
- 显示浏览器窗口便于观察
- 实时日志输出
- 点击后截图验证

使用方法：
```bash
python test_navigation.py
```

## 优化效果预期

## 技术细节

### 选择器策略优先级
1. 精确文本匹配：`:text-is('选品')`
2. 文本包含匹配：`text=选品`
3. 容器+文本：`div:has-text('选品')`
4. 侧边栏特定：`[class*='sidebar'] :text-is('选品')`

### 等待时间调整
| 位置 | 原时间 | 新时间 | 原因 |
|------|--------|--------|------|
| 登录后等待 | 10s | 15s | Echotik首页加载慢 |
| 弹窗检测等待 | 0s | 2s | 给弹窗出现时间 |
| 一级菜单展开 | 800ms | 1200ms | 确保动画完成 |
| 侧边栏加载 | 0s | 2000ms | 新增，确保可交互 |

### 调试信息增强
- 元素数量
- 可见性状态
- 选择器尝试次数
- 页面文本内容分析
- 失败原因详细说明
